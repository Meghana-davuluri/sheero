# reference — Reference Material

This folder holds guides, cheatsheets, and diagrams that RIGGS can reference during sessions. It is not personal context — it is operational knowledge that supports how you work.

---

## Subfolders

| Folder      | Purpose                                               |
|-------------|-------------------------------------------------------|
| `guides/`   | Step-by-step operational guides and cheatsheets       |
| `diagrams/` | Architecture diagrams, system maps, visual references |

---

## How to Use This Folder

- Add guides here for any recurring process you want RIGGS to be able to execute or explain — system backups, deployment steps, troubleshooting flows, etc.
- Diagrams can be referenced by filename — RIGGS won't render them but can direct you to open them
- Keep guides short and action-oriented — they should be executable, not just informational
