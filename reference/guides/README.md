# guides — Operational Cheatsheets

Store step-by-step guides here for any process you run repeatedly. RIGGS can read these during a session to execute or walk you through the steps.

---

## Example Guides to Add

- `git_workflow.md` — your branching and commit conventions
- `backup_procedure.md` — how you back up your system
- `new_project_setup.md` — steps to spin up a new project
- `ops_cheatsheet.md` — tmux, SSH, file locations, common commands

---

## Format

Keep each guide simple:

```
# Guide Title

## Purpose
One line on what this guide is for.

## Steps
1. Step one
2. Step two
3. Step three
```

Executable beats thorough. If RIGGS can read it and do it, it's the right length.
