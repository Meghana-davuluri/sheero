# self-publishing Book Launch Checklist

End-to-end checklist for publishing a new book on self-publishing. Distilled from [pillars/occupational/kdp_strategy.md](../../pillars/occupational/kdp_strategy.md). Follow top to bottom per book.

Last updated: 2026-04-16

---

## 1 — Niche & scope (before building)

- [ ] Niche picked and validated (check BSR of top 3 competitors — ideally all under 100k)
- [ ] Decide format: word search / sudoku / activity / combo / large print
- [ ] Decide audience: kids (ages), adults, seniors, themed
- [ ] Pick trim size (8.5×11 for large print; 6×9 for standard)
- [ ] Set page count target (word search 80–110, sudoku 120–150, big book 200+)

## 2 — Content build

- [ ] Interior generated (puzzles + solutions)
- [ ] One puzzle per page (never two)
- [ ] Font ≥ 18pt for large-print titles
- [ ] Kids books: single-sided printing (mention in description)
- [ ] Cover designed (front + spine + back for paperback; front-only for Kindle)
- [ ] Cover author reads **Pen Name** (standard pen name)

## 3 — Metadata (before hitting publish)

- [ ] Title — top keyword included
- [ ] Subtitle — secondary keywords
- [ ] "Large Print" in *main title* if applicable (not just subtitle)
- [ ] 7/7 keyword slots filled with long-tail phrases (see kdp_strategy.md)
- [ ] 2 primary categories selected in self-publishing dashboard
- [ ] Description — hook + bullets + "Best viewed on tablet or Kindle Fire" for Kindle

## 4 — Pricing

- [ ] Paperback priced per [pricing guide](../../pillars/occupational/kdp_strategy.md#pricing-guide)
- [ ] Kindle eBook at $2.99 (fixed-layout format)
- [ ] Hardcover $14.99–$19.99 where applicable

## 5 — Publish

- [ ] Proof copy ordered OR digital previewer checked on every page
- [ ] Paperback submitted
- [ ] Kindle eBook submitted (fixed-layout)
- [ ] Hardcover submitted (if applicable)

## 6 — Post-publish (within 72 hours of live)

- [ ] Email self-publishing support → request additional categories (up to 10 total)
- [ ] Add book to [pillars/occupational/kdp_portfolio.md](../../pillars/occupational/kdp_portfolio.md)
- [ ] Log revenue tracking start in [pillars/financial/income_tracking.md](../../pillars/financial/income_tracking.md)
- [ ] Note BSR at launch (baseline for future comparison)

## 7 — First month

- [ ] Check BSR weekly
- [ ] Request reviews via "Request a Review" button after first sales
- [ ] If BSR > 200k after 30 days → re-examine keywords and cover, not the content

---

## Red flags — stop and think

- Top competitor BSR > 300k across the niche → niche is dead, pick another
- Can't fit top keyword into title naturally → your title is wrong, not the keyword
- Copying a competitor's cover too closely → A9 penalizes this, and Amazon will pull the listing
